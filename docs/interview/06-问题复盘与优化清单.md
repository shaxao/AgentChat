# 问题复盘与优化清单

## 1. 固定流程导致 Agent 不够自主

### 现象

用户已经明确列出函数、文件、属性、错误点，但系统仍然要求用户继续描述需求；或者任务已经完成后，再追加需求时无法自然进入继续修改。

### 原因

任务执行由固定阶段驱动，Agent 只能在阶段里被动执行。

### 解决

默认改为 Agentic Loop，新建任务、继续修改、运行中插话统一进入自主执行路径。TaskPlan、审查、CI 保留为 guardrail。

### 面试表达

我把流程从“阶段驱动”改成“Agent 决策驱动”。阶段不再决定下一步，Agent 根据上下文、用户输入和验证结果决定下一步。

## 2. “没有文件变更”误判失败

### 现象

冒烟测试、使用说明、诊断类任务可能本来不需要改文件，但系统因为没有文件变更而失败。

### 原因

把“产生文件变更”当成所有子任务的硬成功条件。

### 解决

把文件变更从硬条件降级为信号之一。真正的完成条件是用户目标是否达成、验证是否通过、是否还有失败 guardrail。

## 3. 上下文重复读取导致 token 浪费

### 现象

每轮都读取 `.autocode/MEMORY.md`、`SESSION_SUMMARY.md`、`CI_REPORT.md`、`REVIEW.md` 等文件全文。

### 原因

上下文没有结构化索引，也没有 diff/reconcile。

### 解决

新增 SystemContext Epoch，生成 `SYSTEM_CONTEXT.json`，记录每个上下文源的 hash、summary、priority 和 epoch。

## 4. 构建失败修复效率低

### 现象

构建失败后，Agent 只是跑 `ls`、`glob`，没有真正读取错误输出和对应文件。

### 原因

构建日志没有作为一等上下文，错误输出没有结构化传给 Agent。

### 解决

通过 Tool Output Store 保存完整构建日志，prompt 中放关键错误摘要；失败后进入 Agentic Loop，而不是裸 LLM 随便诊断。

## 5. 中文乱码

### 现象

活动区、日志、PowerShell 输出中出现中文乱码。

### 原因

前后端、PowerShell、服务器日志、Python/Java 输出编码不一致。

### 解决

统一事件 payload 使用 UTF-8，前端展示使用结构化中文摘要，不直接裸显示工具输出；部署脚本和日志读取避免错误编码转换。

## 6. 本地 Runner 连接不稳定

### 现象

Runner 出现 keepalive ping timeout，任务显示本地已连接但工具调用提示未连接。

### 原因

WebSocket 心跳、反向代理超时、服务端连接状态刷新和前端展示状态没有完全统一。

### 解决方向

- Runner 主动重连。
- 服务端维护 session 心跳和 last_seen。
- 任务执行时检查 Runner 可用性。
- 本地导入项目断连时不自动回退云端。
- 反向代理调大 WebSocket timeout。

## 7. 审批状态残留

### 现象

任务已经继续运行或完成，但前端仍显示待确认操作。

### 原因

审批事件、任务事件、前端状态没有以同一个 approval id 做最终一致性更新。

### 解决方向

- 每个 approval request 带唯一 ID。
- approved/rejected/already_resolved 都要发状态收敛事件。
- 任务完成时清理 pending approvals。

## 8. 预览路由和 Next.js 静态导出问题

### 现象

预览页面 `_next/static` 资源 404，a 标签跳转到主系统页面，动态路由缺少 `generateStaticParams()`。

### 原因

Next.js 静态导出部署在 `/autocode-api/workspaces/{id}/preview` 子路径下，资源路径、basePath、动态路由和预览代理需要协同处理。

### 解决方向

- AI 生成 Next.js 项目时提醒静态导出限制。
- 动态路由必须提供 `generateStaticParams()`。
- 预览代理需要处理 `_next/static` 和子路由。
- a 标签内部跳转应保持在 preview base path 下。

## 9. Coding Plan 和用户计费混淆

### 现象

代码开发能使用 Coding Plan 渠道，但普通对话提示套餐余额不足，容易误解为模型问题。

### 原因

上游渠道成本模式和用户侧套餐额度混在一起理解。

### 解决

新增场景化计费策略。Coding Plan 只影响上游成本核算，用户侧是否允许使用由 chat/autocode/workflow 场景额度决定。

## 10. 后续优化清单

- 本地 Runner 增加请求幂等、断点续传、并发限流。
- CacheLedger 增加更细的错误指纹和补丁复用策略。
- Agentic Loop 增加可视化决策轨迹，展示每轮 observe/decide/act/verify。
- 权限系统增加命令 AST 风险识别，而不是只靠字符串规则。
- 预览系统增加框架感知规则：Next.js、Vite、静态 HTML 分别处理。
- 计费系统补管理 UI，让套餐场景额度可视化配置。
