## SSH 配置（推荐）

```bash
# 生成密钥并配置连接复用
.\deploy.ps1 setup-ssh
```

**清理超时时间**：默认 1 小时（3600 秒），可在服务器 B 的 systemd service 文件里加环境变量：

```bash
# /etc/systemd/system/autocode.service
[Service]
Environment="AUTOCODE_PREVIEW_TIMEOUT=3600"  # 1 小时，可按需调整
```

然后 `systemctl daemon-reload && systemctl restart autocode`。



### 1. 🔑 SSH 免密 + 连接复用

**原因**：每次 `ssh`/`scp` 都是独立连接，都要重新认证

**解决**：`deploy.ps1` 新增三个命令：

```powershell
# 一键操作（推荐）：生成密钥 + 配置连接复用
.\deploy.ps1 setup-ssh

# 或分步：
.\deploy.ps1 setup-ssh-key     # 生成密钥并复制到三台服务器
.\deploy.ps1 setup-ssh-config  # 写入连接复用配置
```

**原理**：
- SSH 密钥免密（每台服务器只需输入 1 次密码复制公钥）
- `ControlMaster auto` + `ControlPersist 1h`：第一次连接建立主通道，后续所有 SSH/scp 复用同一通道，1 小时内无需再认证
- 主机别名：`muhugo-a` / `muhugo-b` / `muhugo-c` 替代 IP
