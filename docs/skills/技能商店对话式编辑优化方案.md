# 技能商店对话式编辑优化方案

## 问题描述

用户在使用技能商店时面临以下核心问题：

1. **技能匹配不精准**：用户想要修改某个技能时，难以快速、精准地找到目标技能
2. **操作流程不清晰**：用户不清楚是否需要预先选择agent或开发助手才能修改技能
3. **对话式修改体验差**：无法通过自然语言对话精准地修改技能的各个文件（脚本、配置、文档等）

## 优化目标

1. **实现技能精准匹配机制**：无论用户是否预先选择技能，都能通过对话准确识别目标技能
2. **简化操作流程**：让用户能够便捷地触发技能修改流程
3. **支持完整的技能编辑能力**：通过对话方式修改脚本文件、配置文件、MD文档，以及添加新文件和文件夹
4. **确保AI严格按照用户意图修改**：避免产生无关对话或错误操作

## 技术方案

创建一个智能技能识别器，能够：
- 从用户对话中提取技能名称、描述、关键词

#### 1.2 上下文感知匹配

- 记住用户在对话中提到的技能
- 当用户说"这个技能"、"它"等指代词时，能够关联到之前提到的技能
- 支持"修改刚才那个技能"等自然语言表述

### 2. 对话式技能编辑器

#### 2.1 编辑器触发方式

提供多种触发方式，让用户能够便捷地进入技能编辑模式：

1. **在技能详情页添加"对话式编辑"按钮**
2. **支持在聊天中输入 `/edit-skill <技能名>` 快速触发**
3. **智能识别**：当用户输入类似"修改XX技能"、"编辑XX技能的配置"等表述时，自动触发编辑模式

#### 2.2 编辑器功能

创建一个专门的对话式技能编辑器界面，支持：

**文件树浏览**：
- 显示技能的完整文件结构
- 支持展开/折叠文件夹
- 高亮显示当前正在编辑的文件

**对话式编辑**：
- 用户输入自然语言指令，如"修改SKILL.md中的description字段"
- AI理解用户意图，精准定位到目标文件和目标位置
- 显示修改预览，让用户确认后再应用

**支持的操作**：
- 修改现有文件（脚本、配置、MD文档）
- 添加新文件
- 添加新文件夹
- 删除文件/文件夹
- 重命名文件/文件夹

### 3. 实现细节

#### 3.1 后端API扩展

需要扩展后端API，支持：

1. **获取技能文件树**：`GET /api/v1/agents/:agentId/files`
2. **读取技能文件内容**：`GET /api/v1/agents/:agentId/files/:filePath`
3. **更新技能文件内容**：`PUT /api/v1/agents/:agentId/files/:filePath`
4. **创建新文件/文件夹**：`POST /api/v1/agents/:agentId/files`
5. **删除文件/文件夹**：`DELETE /api/v1/agents/:agentId/files/:filePath`

#### 3.2 前端组件

创建以下前端组件：

1. **SkillEditDialog.tsx**：对话式技能编辑器主界面
2. **SkillFileTree.tsx**：技能文件树组件
3. **SkillEditChat.tsx**：对话式编辑区域组件
4. **SkillEditPreview.tsx**：修改预览组件

#### 3.3 AI助手集成

集成AI助手来理解用户的自然语言指令：

1. **意图识别**：识别用户想要执行的操作（修改、添加、删除等）
2. **文件定位**：根据用户输入精准定位到目标文件
3. **内容修改**：生成修改后的文件内容
4. **修改预览**：向用户展示修改前后的差异

### 4. 用户体验优化

#### 4.1 快速匹配技能

在聊天界面中添加技能快速匹配功能：

1. **输入 `@` 触发技能搜索**：类似@mention功能，输入`@`后显示技能列表
2. **技能名称自动补全**：输入技能名称时提供自动补全建议
3. **最近使用技能**：显示用户最近编辑过的技能，快速访问

#### 4.2 操作流程引导

为新用户提供清晰的操作引导：

1. **首次使用提示**：当用户首次使用技能编辑功能时，显示操作引导
2. **上下文帮助**：在编辑过程中提供上下文相关的帮助信息
3. **错误提示优化**：当匹配不到技能或操作失败时，提供清晰的错误提示和建议

## 关键问题解答

### Q1: 是否一定要选择agent的开发助手才能执行修改技能操作？

**A**: 不需要。优化后的系统支持多种触发方式：
1. 在技能详情页直接点击"对话式编辑"按钮
2. 在聊天中输入 `/edit-skill <技能名>` 快速触发
3. 直接输入自然语言，如"修改XX技能的配置文件"，系统会自动识别并触发编辑模式

### Q2: 如何让用户便捷地知道去修改某个技能？

**A**: 提供多种便捷方式：
1. **技能快速匹配**：输入 `@` 触发技能搜索，类似@mention功能
2. **技能名称自动补全**：输入技能名称时提供自动补全建议
3. **最近使用技能**：显示用户最近编辑过的技能，快速访问
4. **智能识别**：系统能够理解自然语言表述，如"修改刚才那个技能"、"编辑XX技能的配置"等

### Q3: 如何在对话过程中让用户精准、快速匹配到已存在的目标技能？

**A**: 通过以下机制实现：
1. **智能技能识别器**：从用户输入中提取技能相关信息，在技能商店中精准匹配
2. **上下文感知匹配**：记住用户在对话中提到的技能，支持指代词
3. **候选技能列表**：当匹配到多个技能时，向用户展示候选列表让用户选择
4. **技能快速匹配界面**：输入 `@` 触发技能搜索，提供自动补全和最近使用技能
