# 技能商店对话式编辑功能使用指南

## 功能概述

技能商店现在支持**对话式编辑**，您可以通过自然语言描述来修改技能文件，无需手动编辑代码。

## 主要功能

### 1. 对话式编辑

在技能编辑对话框中，切换到"对话式编辑"标签页，然后用自然语言描述您想要做的修改。

**示例命令：**
- "在 SKILL.md 中添加一个新功能说明"
- "修改 scripts/main.py，添加错误处理"
- "创建一个新的配置文件 config.json"
- "更新 README.md，添加安装说明"

### 2. 技能精准匹配

系统会智能识别您想要编辑的技能：
- 支持技能 ID 精确匹配
- 支持技能名称模糊匹配
- 支持描述关键词匹配
- 上下文记忆（可以识别"它"、"这个"等指代词）

### 3. 修改预览

在应用修改之前，系统会显示：
- 需要修改的文件列表
- 每个文件的修改原因
- 新内容的预览
- 修改类型（新建/更新/删除）

### 4. 快捷触发方式

#### 方式一：从技能商店进入
1. 打开技能商店
2. 进入"我的技能"标签页
3. 找到要编辑的技能
4. 点击"编辑文件"按钮

#### 方式二：使用匹配 API
如果您在聊天中提到某个技能，系统可以自动识别并提示您编辑。

## 技术实现

### 后端 API

1. **对话式编辑 API**
   - `POST /api/v1/agent-registry/{agentId}/conversation` - 处理对话式编辑请求
   - `POST /api/v1/agent-registry/{agentId}/conversation/apply` - 应用文件修改

2. **技能匹配 API**
   - `GET /api/v1/agent-registry/match?input={用户输入}` - 从用户输入中匹配技能

### 前端组件

1. **SkillEditDialog** - 技能文件编辑器，支持直接编辑和对话式编辑两种模式
2. **agentRegistryApi** - 添加了 `processConversation`、`applyModifications`、`matchSkill` 方法

### 后端服务

1. **SkillConversationService** - 处理自然语言指令，调用 LLM 理解用户意图
2. **SkillMatchingService** - 智能匹配用户想要编辑的技能
3. **SkillFileManager** - 管理技能文件的 CRUD 操作

## 使用流程

1. **打开技能编辑器**
   - 从"我的技能"页面点击"编辑文件"按钮

2. **选择编辑模式**
   - "直接编辑"：手动编辑文件
   - "对话式编辑"：用自然语言描述修改

3. **对话式编辑流程**
   - 在输入框中描述您想要做的修改
   - 系统调用 LLM 理解您的需求
   - 显示修改建议和预览
   - 点击"应用修改"确认修改

4. **查看修改结果**
   - 修改应用后，文件树会自动刷新
   - 可以切换到"直接编辑"模式查看修改后的文件内容

## 权限控制

- 只有技能的创建者可以编辑该技能
- 管理员可以编辑所有技能
- 其他用户只能查看和安装技能

## 注意事项

1. **备份重要**：在应用大范围修改之前，建议先备份技能文件
2. **审查修改**：仔细审查系统建议的修改，确保其符合您的预期
3. **逐步修改**：对于复杂的修改，建议分多次进行，每次只修改少量内容
4. **测试验证**：修改后务必测试技能是否正常工作

## 常见问题

**Q: 系统无法理解我的需求怎么办？**
A: 尝试用更详细、更清晰的语言描述您的需求。可以提供具体的文件名、函数名等细节。

**Q: 修改后的文件内容不正确怎么办？**
A: 您可以切换到"直接编辑"模式手动修正，或者重新描述您的需求。

**Q: 如何撤销修改？**
A: 目前需要手动恢复文件内容。建议定期备份重要文件。
